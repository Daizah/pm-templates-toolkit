## Purpose

The kickoff meeting officially launches the project, aligns all stakeholders on scope and expectations, and ensures every team member understands their role and responsibilities before work begins.

## Agenda

### 1. Welcome & Introductions (5 min)
- Facilitator opens the meeting
- Brief round of introductions, name, role, and involvement in the project

### 2. Project Overview (10 min)
- Why this project exists and what problem it solves
- How it aligns with organizational goals
- High-level scope summary

**Key questions to answer:**
- What are we building or delivering?
- Why does it matter?
- Who does it impact?

### 3. Scope & Deliverables (10 min)
- Walk through what is in scope and explicitly what is out of scope
- Review key deliverables and what "done" looks like for each
- Confirm acceptance criteria with the client or sponsor

### 4. Roles & Responsibilities (10 min)
- Review who owns what
- Clarify decision-making authority
- Identify escalation path if issues arise

| Role | Owner | Responsibilities |
|---|---|---|
| Project Sponsor | | Final approvals, budget authority |
| Project Manager | | Day-to-day execution, stakeholder updates |
| Team Lead | | Technical delivery, team coordination |
| Client Contact | | Requirements, feedback, sign-off |

### 5. Timeline & Milestones (10 min)
- Walk through the project schedule
- Highlight key milestones and deadlines
- Identify any hard constraints or blackout dates

| Milestone | Target Date | Owner |
|---|---|---|
| Kickoff Complete | | PM |
| Requirements Finalized | | |
| First Draft / Prototype | | |
| Review & Feedback | | |
| Final Delivery | | |
| Project Closeout | | PM |

### 6. Communication Plan (5 min)
- How and how often will status updates be shared?
- Where will project documents live?
- What is the preferred channel for day-to-day communication?

| Type | Frequency | Channel | Owner |
|---|---|---|---|
| Status Update | Weekly | Email / Slack | PM |
| Team Standup | Daily / As needed | Video call | PM |
| Stakeholder Report | Bi-weekly | Email | PM |
| Issue Escalation | As needed | Direct message | PM |

### 7. Risks & Open Questions (5 min)
- Surface any known risks or concerns early
- Capture open questions that need follow-up
- Add items to RAID log after meeting

### 8. Next Steps & Action Items (5 min)
- Confirm immediate next steps for each team member
- Set date for first status check-in
- Confirm where action items will be tracked
